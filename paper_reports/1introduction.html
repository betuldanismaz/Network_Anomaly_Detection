Academic Conference Paper — Section 1

AI-Driven Network Intrusion Detection: A Comparative Study of Traditional Machine Learning and Deep Learning with a Real-Time Production Architecture

Betül Danışmaz  ·  Mustafa Emre Bıyık
Department of Computer Engineering — Network Security & Intelligent Systems Laboratory
Section 1

Introduction & Threat Landscape

The integrity and availability of networked digital infrastructure underpin virtually every domain of modern society — from financial systems and healthcare networks to critical national infrastructure and cloud-hosted enterprise services. As connectivity has expanded, so too has the sophistication, frequency, and economic damage of malicious network activity. According to data aggregated by leading cybersecurity organisations, the global cost of cybercrime exceeded $8 trillion USD in 2023 and is projected to surpass $10.5 trillion annually by 2025 [Cybersecurity Ventures, 2023]. The ability to detect intrusions in real time — before an attacker can exfiltrate data, deploy ransomware, or disrupt services — is therefore not merely an academic exercise but a critical operational imperative.

This section establishes the motivating context for this work. We describe the evolving network threat landscape (§1.1), articulate the specific technical challenges that existing approaches fail to resolve (§1.2), and enumerate the principal contributions of this paper (§1.3).

1.1

The Evolving Network Threat Landscape

A. Rise of Sophisticated, Multi-Vector Cyberattacks

Early network intrusions were characterised largely by opportunistic exploitation of known software vulnerabilities, typically executed by individual actors using single-technique attack vectors. The contemporary threat landscape is fundamentally different in both scale and complexity. Modern adversaries — including nation-state actors, organised cybercriminal syndicates, and Advanced Persistent Threat (APT) groups — conduct multi-phase, multi-vector campaigns that combine volumetric flooding, stealthy reconnaissance, and semantic application-layer exploitation within the same intrusion sequence.

The CICIDS2017 dataset, which forms the empirical foundation of this work, captures this multi-vector reality by generating and labelling five distinct categories of real-world attack traffic over five days of network operation. These categories include Distributed Denial-of-Service (DDoS) floods, Denial-of-Service (DoS) variants (Slowhttptest, Slowloris, Hulk, GoldenEye), Botnet command-and-control communications, Brute Force credential attacks, Port Scanning, Web Attacks (SQL Injection, Cross-Site Scripting, Cross-Site Request Forgery), and Infiltration scenarios. A particularly important characteristic of the modern threat environment is its asymmetric class distribution. In the CICIDS2017 dataset, as processed in this work, benign traffic constitutes approximately 80% of 2.83 million total flows, while Volumetric attack flows constitute approximately 14%, and the operationally critical Semantic attack class — comprising the hardest-to-detect exploits — accounts for only ~6%. This imbalance is not an artifact of the dataset but a realistic property of enterprise network traffic that any practical NIDS must handle.

B. Limitations of Signature-Based and Rule-Based NIDS

Traditional Network Intrusion Detection Systems (NIDS) rely on one of two fundamental detection paradigms: signature-based detection, which matches observed network events against a database of known attack patterns, and rule-based anomaly detection, which flags traffic that deviates from manually specified thresholds and protocol expectations. Representative systems include Snort, Suricata, and commercial equivalents that dominate enterprise deployments.

⚠ Critical Limitation

Signature-based NIDS are fundamentally reactive. They can only detect attacks whose signatures have already been catalogued. Against zero-day vulnerabilities, polymorphic malware, and novel attack variants — precisely the threats that pose the greatest operational risk — they are, by design, blind. Maintaining a current signature database requires continuous expert effort and introduces latency between the discovery of new attacks and their detectability in deployed systems.

Rule-based systems suffer from complementary but equally severe limitations. Manually tuning detection thresholds for every network environment is labour-intensive and error-prone. Static rules cannot adapt to concept drift — the evolution of traffic patterns over time as user behaviour, applications, and attacker tactics change. Furthermore, rules designed for one network topology typically require substantial rework before deployment in a different environment. Both paradigms share a fundamental inability to generalise: they encode past knowledge but cannot learn from data.

C. The Case for AI/ML-Driven Anomaly Detection

Machine learning and deep learning offer a fundamentally different detection philosophy: rather than cataloguing known bad behaviour, these approaches learn a statistical model of network traffic from labelled examples and classify new flows based on their proximity to learned boundaries in a high-dimensional feature space. This data-driven approach confers several decisive advantages over the paradigms described above.

At the feature representation level, modern flow-level feature extraction tools such as CICFlowMeter generate rich, bidirectional flow descriptors — including packet length statistics, inter-arrival time distributions, flag counts, and byte rate ratios — that are largely invariant to payload encryption and expose behavioural signatures that are difficult for attackers to disguise. At the model architecture level, gradient-boosted tree ensembles such as XGBoost and ensemble methods such as Random Forest have demonstrated state-of-the-art performance on tabular flow features, while recurrent deep learning architectures such as LSTM and BiLSTM can additionally exploit the temporal ordering of flows within a session — a dimension entirely absent from tabular models — to detect attacks whose signatures span multiple consecutive network exchanges.

💡 Core AI Advantage

Unlike signature-based systems that compare individual packets to a fixed catalogue, ML/DL models learn latent statistical patterns across a high-dimensional feature space. A well-trained model can detect not only attacks it has seen during training, but also behavioural deviations that share structural similarity with known threats — providing a meaningful degree of generalisation to novel attack variants.

The deployment of AI-driven NIDS is further motivated by the scale of modern network traffic. Enterprise networks routinely handle millions of flows per hour. The XGBoost model trained in this work achieves an inference throughput of 126,546 samples/second at a latency of 0.008 ms/sample, demonstrating that machine learning classifiers can operate at network line rate without introducing unacceptable latency into the detection pipeline.

1.2

Problem Statement

Despite the compelling theoretical advantages of AI-driven NIDS and an extensive published literature reporting high classification accuracy on benchmark datasets, practical deployment of ML-based intrusion detection in production environments remains surprisingly rare. We identify three interconnected technical challenges that account for this gap, and which directly motivate the design choices of the system presented in this paper.

⚖️
Challenge 1: Severe Class Imbalance

In realistic network traffic, attack flows constitute a small minority of total observations. In CICIDS2017 as processed in this work, the hardest-to-detect Semantic attack class represents only ~6% of 2.83M total flows. Standard classifiers trained on imbalanced data are systematically biased toward the majority (Benign) class, producing inflated aggregate accuracy while failing precisely on the attack classes most in need of detection. Addressing this requires explicit class-weighting strategies, careful metric selection (macro F1 over accuracy), and stratified sampling protocols that preserve class proportions in every data partition.

⏱️
Challenge 2: Temporal Data Leakage

The CICIDS2017 dataset spans five days of traffic capture in separate per-day CSV files. A naive random shuffle-and-split procedure mixes flows from the same time window across train and test sets, allowing the model to effectively memorise the temporal context of attacks before evaluation. This inflates reported test accuracy by an amount that is indeterminate but potentially substantial. Our Per-File Stratified Split methodology is explicitly designed to prevent this leakage by treating each source file as an independent, temporally cohesive unit.

⚡
Challenge 3: Real-Time Inference Latency

A NIDS that cannot classify traffic flows at the speed of network ingestion provides no operational value. Deep learning models — particularly sequential models applied to variable-length packet sequences — can exhibit inference latency inconsistent with real-time deployment requirements. The system presented in this paper benchmarks all models on inference latency, with XGBoost achieving 0.008 ms/sample throughput of 126,546 samples/sec, establishing a practical upper bound for production deployment.

🏭
Challenge 4: The Research-to-Production Gap

The overwhelming majority of NIDS literature evaluates models in an offline, static-dataset setting. The real-world deployment challenge — integrating a trained model into a live packet capture pipeline, handling streaming data with variable arrival rates, managing model updates without service interruption, and triggering automated responses to detected threats — receives comparatively little attention. This work addresses that gap through a fully operational, Kafka-backed production architecture described in §6.

The Performance-Deployability Trade-Off

A critical and under-examined tension in AI-driven NIDS research is the trade-off between classification performance and deployment feasibility. Deep learning models — particularly BiLSTM networks — often achieve marginally higher detection accuracy on specific attack classes by modelling temporal dependencies in flow sequences. However, they require three-dimensional input construction (sliding-window sequencing), significantly longer training times (2–3 hours for BiLSTM vs. ~20 seconds for GPU-accelerated XGBoost), larger model files (3.9 MB for BiLSTM vs. 13.2 MB for XGBoost at a different tradepoint), and exhibit higher inference latency relative to gradient-boosted trees. Conversely, XGBoost achieves highly competitive overall accuracy (97.71%) and macro F1 (95.87%) with sub-millisecond per-sample inference — making it uniquely suitable for deployment at network line rate.

📌 Central Research Question

Under leakage-free, class-imbalance-aware evaluation conditions, do deep sequential models (LSTM, BiLSTM) provide a statistically meaningful accuracy advantage over traditional gradient-boosted tree ensembles (XGBoost, Random Forest) on a realistic 3-class network intrusion detection task — and if so, does that advantage justify the additional inference latency and engineering complexity in a production deployment context?

The Offline-to-Online Deployment Divide

When a machine learning model trained on a static labelled dataset is deployed to operate on live network traffic, it encounters a fundamentally different data distribution. Real network flows arrive as a continuous, high-throughput stream; they must be windowed, feature-extracted, scaled using training-time statistics, and classified at low latency — all in real time, without access to the ground-truth labels available during evaluation. The transition from offline benchmark to production deployment requires solving a suite of engineering problems that the academic literature routinely omits: feature alignment between training-time and inference-time feature extractors, dynamic model loading and hot-swapping, asynchronous message queuing to handle traffic bursts, and automated response integration. The Live Bridge Mechanism designed in this work addresses each of these requirements explicitly.

1.3

Contributions of This Work

In response to the challenges identified in §1.2, this paper makes the following six principal contributions to the fields of network security and applied machine learning:

  • C1 Per-File Stratified Split — A Leakage-Free Evaluation Protocol. We introduce a preprocessing methodology that applies stratified train/validation/test partitioning independently to each source file of the CICIDS2017 dataset before any cross-file merging. This design provides a mathematical guarantee that no temporal context crosses the train/test boundary, eliminating the data leakage that inflates accuracy metrics in naive random-split protocols used by the majority of published NIDS studies. Additionally, within-file stratification ensures that minority attack classes — particularly the Semantic class at ~6% prevalence — are proportionally represented in the test partition despite the global imbalance, preventing the pathological case where rare attacks are absent from the evaluation set entirely. This contribution is described formally in §3.4.
  • C2 Operational 3-Class Attack Taxonomy — Bridging Research and SOC Triage. We propose a principled 3-class labelling scheme — Benign, Volumetric (DDoS, DoS, Botnet), and Semantic (Port Scan, Web Attack, Brute Force, Infiltration) — designed to align detection output with the triage requirements of a real-world Security Operations Centre (SOC). Binary Benign/Attack formulations discard the operationally meaningful distinction between resource-exhaustion attacks (requiring network-level countermeasures) and application-layer exploits (requiring endpoint investigation and patch management). Our 3-class taxonomy preserves this distinction while avoiding the data-sparsity pathologies of fine-grained per-attack labelling with 15+ classes. The taxonomy is formally defined in §3.2.
  • C3 Systematic ML vs. DL Comparative Study with Identical Evaluation Conditions. We conduct a rigorous, reproducible comparative evaluation of five model architectures — XGBoost (GPU-accelerated), Random Forest, Decision Tree, LSTM, and Bidirectional LSTM — trained under identical data partitions, class-weighting strategies, and evaluation protocols. Crucially, all models are evaluated against the same leakage-free test set produced by the Per-File Stratified Split (C1), making our comparative results directly interpretable and reproducible. Comparisons span classification performance (accuracy, macro Precision, Recall, F1, ROC-AUC), inference latency (ms/sample), training time, and model artifact size — providing a multi-dimensional view of the performance-deployability trade-off. AutoML benchmarking via PyCaret across 10+ algorithm families is additionally employed to situate our hand-tuned models within a comprehensive baseline landscape. Results are reported in detail in §8.
  • C4 Production-Grade Live Bridge Mechanism — From Packet to Prediction. We design and implement an end-to-end real-time inference pipeline that integrates: (i) live packet capture via Scapy in 4-second sliding windows; (ii) bidirectional flow feature extraction via CICFlowMeter, reducing 78 raw CICFlowMeter features to 20 selected features; (iii) an Apache Kafka-backed message queue (Docker-containerised) that decouples traffic ingestion from inference and handles burst conditions; (iv) a Kafka Consumer that performs model-specific preprocessing (2D tabular or 3D temporal sequencing), inference, and alert generation; and (v) an automated Firewall Response Engine that executes OS-level IP blocking upon confirmed attack classification. The system additionally supports zero-downtime model hot-swapping — switching between any of the five trained models at runtime without restarting the inference service — via a shared configuration file and a consumer polling mechanism. System architecture is detailed in §6.
  • C5 Integrated Attack Simulation Environment — Dynamic Closed-Loop Validation. We develop a controlled attack simulation environment — comprising TCP SYN flood scenarios using hping3 for Volumetric attack validation and port scanning using Nmap for Semantic attack validation — that is coupled directly to the Live Bridge Mechanism. This closed-loop configuration enables dynamic model validation under simulated adversarial conditions that complement the static offline evaluation on the CICIDS2017 test set, measuring end-to-end detection latency from packet ingestion to automated alert generation. This contribution provides a validation methodology that more closely approximates real operational conditions than offline test-set evaluation alone, and is described in §6.3.
  • C6 End-to-End MLOps Pipeline with AutoML Benchmarking and GPU Acceleration. The complete system is developed with engineering practices drawn from the MLOps discipline: reproducible, seeded data splits; versioned model artifacts (.pkl, .keras); config-driven experiment management; automated evaluation dashboards with per-class confusion matrices, ROC curves, and feature importance rankings; and a real-time monitoring dashboard built with Streamlit. GPU acceleration via CUDA reduces XGBoost training convergence time by a factor of 15–45× over CPU-only baselines. TensorFlow/Keras cuDNN-optimised LSTM kernels are used for deep learning model training. The pipeline is described in §7.
✅ Positioning Statement

Taken together, these six contributions advance the state of the art in AI-driven NIDS research on two complementary dimensions: methodological rigour — through a leakage-free evaluation framework and a multi-dimensional comparative study that establishes reproducible, interpretable performance benchmarks — and engineering completeness — through a production-grade real-time architecture that closes the gap between offline benchmark evaluation and operational deployment in live network environments.
