Section 10

Conclusion & Future Work

This paper presented a comprehensive, end-to-end AI-driven Network Intrusion Detection System that spans rigorous offline benchmarking, a novel 3-class attack taxonomy, and a production-grade real-time inference pipeline. We close with a synthesis of contributions, headline results, future research directions, and the broader implications of this work.

10.1

Conclusion

We have presented a systematic study comparing five machine learning models — XGBoost, Random Forest, Decision Tree, LSTM, and BiLSTM — for 3-class network intrusion detection on the CICIDS2017 benchmark dataset, evaluated under a mathematically rigorous, leakage-free experimental protocol, and deployed in a production-grade real-time inference pipeline. The study's six core contributions are summarised below:

  • C1 Per-File Stratified Split: A temporal-aware data partitioning methodology that independently stratifies each of the 8 CICIDS2017 source files, eliminating temporal data leakage while preserving minority class representation across train/val/test splits. Verified by an 8-check automated data integrity audit.
  • C2 3-Class Attack Taxonomy: A novel operationally-motivated attack taxonomy (Benign / Volumetric / Semantic) that collapses 15 raw CICIDS2017 labels into interpretable SOC-actionable categories, addressing the fragmentation problem of existing multi-label benchmarks while preserving attack-response relevance.
  • C3 Systematic ML vs DL Comparative Study: A rigorous 5-model comparison — traditional ML (XGBoost, RF, DT) vs deep learning (LSTM, BiLSTM) — on identical features, identical test sets, and identical evaluation metrics. Key finding: the DL–ML Macro F1 gap is only 0.58 pp (LSTM: 96.45%, XGBoost: 95.87%), with XGBoost achieving 126,546 samples/sec at 0.008 ms/sample — 37× faster than LSTM.
  • C4 Live Bridge Production Architecture: A 15-component, loosely-coupled microservice pipeline (Scapy → CICFlowMeter → Kafka → AI Consumer) that enables real-time flow feature extraction, 3-class inference, and IP-level automated firewall response — deployable with a single docker-compose up -d command.
  • C5 Zero-Downtime Model Hot-Swap: A configuration file–driven mechanism that enables runtime switching between any of the six trained models (XGBoost, RF, DT, LSTM, BiLSTM, PyCaret champion) without restarting the Kafka consumer or capture pipeline, enabling direct A/B model comparison against live traffic.
  • C6 MLOps Engineering Stack: An end-to-end reproducibility and observability stack including: SHAP TreeExplainer (runtime attack explanations), automated data integrity auditing, config-paired model artifacts, Docker-containerised Kafka infrastructure, automated report generation (9 artefact types × 5 models), and an async traffic harvesting pipeline for continual retraining.

Three findings are particularly noteworthy for the research community: (1) the unidirectional LSTM outperforms the bidirectional BiLSTM on this task, suggesting that bidirectionality should not be assumed superior for flow-based sequential classification; (2) the Decision Tree achieves the highest Semantic F1 (97.53%) — above all deep learning models — demonstrating that simple, interpretable models retain significant value for attack classes with rule-separable flow signatures; and (3) the performance gap between the best traditional ML model (XGBoost) and the best DL model (LSTM) is only 0.58 percentage points, while XGBoost achieves a 37× throughput advantage, making it the clear recommendation for real-time production deployment.

📌 Central Thesis

In modern AI-driven NIDS, the choice between traditional ML and deep learning is not a question of which model family is categorically superior, but which model best satisfies the specific deployment constraint: Macro F1 fidelity favours LSTM, real-time throughput favours XGBoost, minority-class precision favours Decision Tree, and operational interpretability requires all three — unified under a hot-swappable production architecture such as the Live Bridge.

10.2

Future Work

The following research directions are directly motivated by the limitations identified in §9.3 and the open questions raised by the experimental findings. Items are categorised by estimated implementation horizon.

🟢 Near-term (0–6 months)
Proper 3-Way DL Split
Implement a dedicated 70/15/15 train/validation/test split for LSTM and BiLSTM to eliminate the test-set early stopping bias documented in §9.1. Report corrected Macro F1 to quantify the bias magnitude.
🟢 Near-term (0–6 months)
OvR ROC-AUC for DL Models
Extend the evaluation scripts (evaluate_lstm.py, evaluate_bilstm.py) to compute calibrated OvR ROC-AUC scores using the softmax output probabilities, enabling a complete cross-model comparison on this metric.
🟢 Near-term (0–6 months)
Feature Ablation Study
Use the existing SHAP explainer (xai_engine.py) to systematically ablate Top-K feature subsets (K=5, 10, 15, 20, 30) and measure the impact on Macro F1 and Semantic F1 for each model. Determine the minimum feature set for production deployment.
🟢 Near-term (0–6 months)
Live Attack Simulation Quantification
Execute the full attack simulation protocol (Table 14) against the Live Bridge and report end-to-end detection latency, live false positive rate (30-min benign baseline), and per-attack live detection rate for each of the 8 attack scenarios.
🔵 Medium-term (6–18 months)
Cross-Dataset Evaluation
Train and evaluate all five models on CICIDS2018, UNSW-NB15, and CIC-DDoS2019 using the same per-file stratified split methodology. Measure cross-dataset Macro F1 degradation to quantify the generalisation gap and identify which model architectures transfer most robustly.
🔵 Medium-term (6–18 months)
eBPF / DPDK Line-Rate Feature Extraction
Replace the CICFlowMeter CLI subprocess (which dominates pipeline latency at 10–90 seconds) with a kernel-level eBPF flow exporter or DPDK-based flow meter that can compute equivalent bidirectional flow statistics at line rate. This would unlock true per-flow real-time inference using LSTM.
🔵 Medium-term (6–18 months)
Adversarial Robustness Evaluation
Evaluate all models against adversarial flow perturbations — minimal feature modifications (FGSM-style) designed to evade detection while preserving attack effectiveness. Assess the robustness differential between tree-based and neural models, and implement adversarial training warm-up for DL models.
🔵 Medium-term (6–18 months)
Continual Learning from Live Traffic
Leverage the live traffic harvesting pipeline (§6.6) to collect analyst-labelled flows from production deployment. Implement online fine-tuning protocols for XGBoost incremental trees and LSTM adapter layers to adapt models to local traffic distributions without full retraining.
🟣 Long-term (18+ months)
Graph Neural Network for Flow Correlation
Model network traffic as a temporal flow interaction graph, where nodes are hosts and edges are flows. Apply Graph Attention Networks (GAT) to classify flows based not only on their own features but on their structural position in the network graph — enabling detection of coordinated multi-flow attacks invisible to per-flow classifiers.
🟣 Long-term (18+ months)
Foundation Model Adaptation (LLM for NetFlow)
Investigate the adaptation of sequence foundation models (e.g., pre-trained Transformers) to network flow token sequences, analogous to CodeBERT for source code. Pre-train on large-scale unlabelled NetFlow data and fine-tune on the 3-class taxonomy — exploring whether self-supervised flow representations outperform the supervised LSTM/BiLSTM trained from scratch.
🟣 Long-term (18+ months)
Federated NIDS for Privacy-Preserving Collaboration
Design a federated learning protocol where multiple organisations train local NIDS models on their private traffic without centralising raw flows. Aggregate model updates using federated averaging to produce a global model that benefits from diverse attack exposure while satisfying data sovereignty constraints.
🟣 Long-term (18+ months)
Zero-Day Anomaly Detection Integration
Augment the supervised 3-class classifier with an unsupervised anomaly detection layer (e.g., Isolation Forest or Variational Autoencoder on flow embeddings). Flows classified as outliers by the anomaly detector but as "Benign" by the classifier trigger a human-review escalation — creating a hybrid system capable of flagging novel attacks unseen in CICIDS2017 training data.
✅ The Road Ahead — Research Continuity

The infrastructure built in this work — the Per-File Stratified Split methodology, the Live Bridge pipeline, the async TrafficLogger, the hot-swap consumer, and the SHAP XAI engine — is specifically designed to support the future work directions listed above. The codebase is publicly available at github.com/betuldanismaz/Network_Anomaly_Detection and is structured to enable direct extension by the research community without requiring redesign of the core pipeline.

Paper Summary — AI-Driven NIDS: End-to-End Contributions

This work demonstrates that a production-viable AI-driven NIDS requires simultaneous excellence across three dimensions that much of the existing literature treats in isolation: methodological rigour (per-file stratified split, leakage-free evaluation, 8-check data audit), predictive performance (Macro F1 up to 96.45%; ROC-AUC up to 0.9991; Semantic F1 up to 97.53%), and production readiness (126,546 samples/sec, zero-downtime hot-swap, SHAP-attributed alerts, automated firewall response, async traffic harvesting for continual improvement).

The 0.58 pp Macro F1 gap between the best traditional ML model (XGBoost: 95.87%) and the best deep learning model (LSTM: 96.45%) — measured under a consistent, leakage-free protocol — challenges the common narrative of categorical DL superiority in network intrusion detection. For real-time production deployment, XGBoost emerges as the Pareto-optimal choice: near-top Macro F1, best ROC-AUC, and 37× higher throughput. LSTM is preferred for accuracy-maximising offline analysis. Decision Tree offers the most interpretable and surprisingly competitive option for Semantic-class detection.

We release all training scripts, preprocessing pipelines, evaluation dashboards, and the complete Live Bridge architecture to support reproducibility and future replication — addressing the reproducibility crisis in AI-driven NIDS research identified in §2.5.
