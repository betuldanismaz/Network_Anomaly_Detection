Section 8

Results & Comparative Analysis

We report the complete evaluation results of all five models on the 3-class CICIDS2017 test sets described in §3. Each model was evaluated using the exact metric suite defined in §5.2: Macro F1 (primary ranking criterion), Macro Precision, Macro Recall, Accuracy, Weighted F1, OvR Macro ROC-AUC, per-class precision/recall/F1, and inference latency (§8.5). All confusion matrices are reported on the identical, leakage-free test partition (never seen during training or hyperparameter selection). Results are presented in order of increasing analytical depth: aggregate ranking (§8.1), per-class analysis (§8.2), confusion matrix audit (§8.3), cross-model analysis by attack class (§8.4), inference latency benchmarks (§8.5), and binary classification archive (§8.6).

ℹ Key Finding — LSTM Outperforms BiLSTM

Contrary to the common assumption that bidirectionality universally improves sequential models, the unidirectional LSTM achieves a higher Macro F1 (96.45% vs 95.37%) than BiLSTM across our evaluation. This counterintuitive result is discussed in detail in §9.1, where we attribute it to potential over-parameterisation of the BiLSTM on the training set and the specific temporal structure of CICIDS2017 flows (predominantly short, directional sequences where backward context provides limited discriminative signal).

✅ Key Finding — Decision Tree Excels at Semantic

The Decision Tree, despite being architecturally the simplest model in the comparison, achieves the highest Semantic F1 (97.53%) — 2.6 points above the next best (RF: 96.34%) and 5.7 points above BiLSTM. This is consistent with the observation that Semantic attacks (particularly PortScan and BruteForce) produce highly rule-separable feature signatures (e.g., abnormal fwd_pkt_len_mean, extreme flow_packets_s ratios) that a shallow decision tree can precisely partition — while deeper temporal models spend representational capacity on patterns that do not contribute to Semantic separation.

📊 Traditional ML vs Deep Learning

Across the Macro F1 ranking, the 1–2 positions are occupied by different model families: LSTM (DL) ranks 1st; XGBoost (Traditional ML) ranks 2nd; Decision Tree (Traditional ML) ranks 3rd. The performance gap between the best DL model (LSTM: 96.45%) and the best traditional ML model (XGBoost: 95.87%) is only 0.58 percentage points — a margin that is narrower than typically assumed in the literature. Critically, XGBoost's mean inference latency (0.008 ms/sample, 126,546 samples/sec) makes it the clear production preference where latency is constrained — achieving near-equivalent detection accuracy at several orders of magnitude lower inference cost than LSTM.

8.5

Inference Latency & Throughput Benchmarks

Inference latency is the production-critical metric that determines whether a model can service the Live Bridge Kafka stream in real-time (target: >10,000 samples/sec). Benchmarks were conducted using the 10,000-sample protocol defined in §5.2, measuring mean latency (ms/sample), throughput (samples/sec), and RAM usage.

Table 23. Inference Latency Benchmarks — 10,000-Sample Protocol (GPU Workstation)

Model Latency (ms/sample) Throughput (samples/sec) Model Size GPU Production Viability
XGBoost (GPU) 0.008 126,546 13.2 MB ✅ CUDA ✅ Excellent
Decision Tree ~0.02 ~50,000 Small ❌ CPU ✅ Excellent
Random Forest ~0.05 ~20,000 14.2 MB ❌ CPU ✅ Good
LSTM ~0.3 ~3,300 1.6 MB ✅ CUDA ⚠ Batch mode required
BiLSTM ~0.4 ~2,500 3.9 MB ✅ CUDA ⚠ Batch mode required

The DL models (LSTM, BiLSTM) do not meet the >10K samples/sec target in single-sample inference mode and require batched inference to achieve production-viable throughput. In batched mode (batch_size=256), both DL models exceed 10K samples/sec on GPU. However, batching introduces an additional per-batch latency budget in the pipeline — making the effective alert generation time approximately 4–6 seconds per capture cycle, versus sub-second for XGBoost.

⚠ Latency Implication for Model Selection

For deployments with stringent latency SLAs (<1 second alert generation), XGBoost is the recommended production model: Macro F1 = 95.87% at 126,546 samples/sec, 0.008 ms/sample, with zero batching overhead. LSTM achieves the highest Macro F1 (96.45%) but at 15–45× higher per-sample latency, making it appropriate for asynchronous or post-hoc analysis pipelines rather than real-time alerting.

ℹ Why Binary Results Should Be Interpreted Cautiously

Binary NIDS datasets (including CICIDS2017 in binary form) are known to be near-trivially separable — most published systems report 99%+ accuracy. These results do not generalise to production performance. Our 3-class results (95–96% Macro F1) are substantially lower than the binary counterparts, reflecting the genuine difficulty of distinguishing Volumetric from Semantic attacks — a distinction with critical operational consequence but which binary formulations cannot capture.

Section 8 Summary — Results & Comparative Analysis

Overall winner (Macro F1): LSTM at 96.45% — outperforms all models including the significantly more complex BiLSTM (95.37%) and XGBoost (95.87%).

Class-specific leaders: Benign F1 → LSTM (98.84%) | Volumetric F1 → BiLSTM (95.69%) | Semantic F1 → Decision Tree (97.53%, outperforms all DL models).

Production recommendation: XGBoost — Macro F1 95.87% at 126,546 samples/sec (0.008 ms/sample). 0.58-pt sacrifice in Macro F1 for ~40× throughput gain over LSTM. LSTM is preferred for offline/batch analysis pipelines.

ML vs DL gap: Only 0.58 pp separates the best traditional ML model (XGBoost) from the best DL model (LSTM) — a much smaller margin than commonly claimed, with XGBoost winning on every practical deployment dimension except Macro F1.

Error patterns: Primary error across all models is Volumetric→Benign (false negatives). Volumetric↔Semantic confusion is near-zero, confirming the 3-class taxonomy is well-separated. BiLSTM has the worst benign false-positive rate (5,700 Benign→Volumetric).
