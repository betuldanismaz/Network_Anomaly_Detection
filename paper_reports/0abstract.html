AI-Driven Network Intrusion Detection: A Comparative Study of Traditional Machine Learning and Deep Learning with a Real-Time Production Architecture

Betül Danışmaz  ·  Mustafa Emre Bıyık
Department of Computer Engineering — Network Security & Intelligent Systems Laboratory
Network Intrusion Detection Deep Learning / LSTM MLOps / Real-Time Inference CICIDS2017 XGBoost · Random Forest Apache Kafka

Abstract

The rapid proliferation of sophisticated, multi-vector cyber threats has rendered traditional signature-based and rule-based Network Intrusion Detection Systems (NIDS) increasingly inadequate for protecting modern enterprise environments. Contemporary attacks — including volumetric denial-of-service floods, stealthy semantic exploits such as SQL injection and cross-site scripting, and coordinated botnet campaigns — demand detection systems capable of learning complex, high-dimensional traffic patterns and making accurate, low-latency classification decisions at network line rate. Although a rich body of machine learning literature addresses offline classification accuracy on benchmark datasets, a critical and persistent gap separates academic evaluation from deployable production systems: the joint challenges of temporal data leakage, severe class imbalance, and the architectural disconnect between offline model training and real-time inference remain insufficiently addressed in the existing literature.

This paper presents a comprehensive, end-to-end AI-driven NIDS designed to close that gap through four principal contributions. First, we introduce a Per-File Stratified Split methodology applied to the CICIDS2017 dataset — a 2.83-million-flow corpus spanning five days of realistic network traffic — which isolates each daily capture file before performing stratified train/validation/test partitioning. This design mathematically eliminates the temporal leakage that arises when flows from the same time window appear in both training and test sets, a flaw endemic to naive random-split protocols and one that artificially inflates reported generalisation metrics in prior work.

Second, we propose a novel operationally motivated 3-class attack taxonomy — Benign, Volumetric (encompassing DDoS, DoS variants, and Botnet traffic), and Semantic (encompassing Port Scanning, Web Attacks including SQL Injection and XSS, Brute Force, and Infiltration) — that aligns detection granularity with real-world incident response workflows. Unlike binary normal/attack formulations that discard actionable threat intelligence, or fine-grained per-attack labellings that suffer from data-sparsity pathologies, our 3-class design enables triage-level classification while maintaining sufficient signal density across all classes during training.

Third, we conduct a rigorous and reproducible comparative study between two paradigmatically distinct model families. On the traditional machine learning side, we train and evaluate XGBoost (with GPU CUDA acceleration via tree_method=hist) and Random Forest, both operating on two-dimensional tabular flow feature vectors comprising 20 CICFlowMeter-derived features selected through importance ranking. On the deep learning side, we train Long Short-Term Memory (LSTM) and Bidirectional LSTM (BiLSTM) networks on three-dimensional temporal sequences constructed via a sliding-window protocol (window length = 10, stride = 1), enabling the models to exploit inter-flow temporal dependencies that tabular models cannot capture. Additionally, AutoML benchmarking via PyCaret across more than ten algorithm families is employed to establish a comprehensive comparative baseline.

Our experimental results demonstrate that all five trained models achieve 3-class classification accuracy in the range of 97.13%–98.15%, with macro F1-scores spanning 95.37%–96.45%. The LSTM model achieves the highest overall accuracy (98.15%) and macro F1-score (96.45%), while XGBoost offers a compelling accuracy-vs-latency trade-off, achieving 97.71% accuracy with a mean inference latency of only 0.008 ms/sample and a throughput of 126,546 samples/second — suitable for deployment at network line rate. ROC-AUC scores exceed 0.998 across all traditional ML models, confirming strong probabilistic discrimination across all three classes.

Fourth, we describe the design and validation of a production-grade Live Bridge Mechanism that integrates real-time packet capture via Scapy, bidirectional flow feature extraction via CICFlowMeter, and an Apache Kafka-backed streaming pipeline that decouples traffic ingestion from inference. The Kafka Consumer supports dynamic hot-swapping between all trained models without service interruption, enabling seamless in-production model comparison and operational flexibility. A companion Attack Simulation Environment — exercising representative Volumetric (TCP SYN flood via hping3) and Semantic (port scanning via Nmap) attack scenarios — is used to validate the integrated system under controlled conditions that mirror real adversarial behaviour, measuring end-to-end detection latency from packet ingestion to alert generation. An automated Firewall Response Engine issues OS-level IP blocking rules upon confirmed attack classification, closing the loop from detection to active defence.

The complete system — encompassing data pipeline, model training, evaluation, streaming infrastructure, and monitoring dashboard — is built with MLOps engineering practices including versioned model artefacts, seeded reproducible splits, config-driven experiment management, and a Streamlit-based real-time monitoring dashboard. GPU-accelerated training reduces XGBoost convergence time by a factor of 15–45× over CPU-only baselines. This work contributes both a methodologically sound evaluation framework for future NIDS research and a reference architecture for deploying AI-driven intrusion detection in operational network environments.

Summary of Key Contributions

  • C1
    Per-File Stratified Split: A novel preprocessing protocol applied to CICIDS2017 that eliminates temporal data leakage and guarantees proportional representation of minority attack classes (Semantic: ~6% of traffic) across all training and evaluation partitions.
  • C2
    Operational 3-Class Taxonomy: A principled attack grouping (Benign / Volumetric / Semantic) that bridges the gap between academic classification granularity and the triage requirements of real-world Security Operations Centre (SOC) workflows.
  • C3
    Systematic ML vs. DL Comparative Study: Controlled experimental comparison of XGBoost, Random Forest, Decision Tree, LSTM, and BiLSTM under identical data partitions and evaluation protocols, augmented with AutoML (PyCaret) benchmarking across 10+ algorithms.
  • C4
    Production-Grade Real-Time Architecture: An end-to-end Kafka-based streaming pipeline integrating live packet capture, real-time feature extraction, dynamic model hot-swapping, automated firewall response, and a monitoring dashboard — validated under simulated adversarial conditions.

Significance Statement

This work addresses a critical reproducibility and deployment gap in AI-driven network security research by simultaneously contributing a methodologically rigorous evaluation framework — centred on leakage-free, stratified temporal partitioning — and a fully operational, streaming-capable reference architecture that bridges the divide between offline benchmark performance and real-world production deployment. Our results demonstrate that with appropriate preprocessing discipline, both traditional gradient-boosted tree ensembles and deep sequential models can achieve macro F1-scores exceeding 95% on a realistically imbalanced 3-class threat taxonomy derived from CICIDS2017, while the proposed Live Bridge Mechanism enables sub-200 ms end-to-end detection latency from raw packet ingestion to automated firewall response — a target directly relevant to real-time SOC operations.
